# 网站内容分类规范（2026-07-11 定）

> 发新文章时照此填 front-matter，不要发明新词。目录带下划线，Jekyll 不会构建本文件。

## categories：两级，各从固定词表选一

**一级 = 内容形态（三选一）**

| 值 | 收什么 |
|---|---|
| 概念 | 讲清一个方法"是什么、怎么用"（如 5S 指南、VSM 绘制、五原则） |
| 思考 | 观点判断、行业洞察、影视精益课、找茬系列 |
| 实践 | 客户现场与落地复盘、脱敏案例（含数据与前后对比） |

**二级 = 主题域（四选一）**

| 值 | 收什么 |
|---|---|
| 精益基础 | TPS、价值流、浪费、精益思想等根基话题 |
| 现场管理 | 5S、变化点、层级会议、班组、质量、现地现物 |
| 战略执行 | 方针管理、指标树、目标分解、战略落地 |
| 数字化与AI | 精益数字化、AI 落地、数据治理、智能制造 |

写法：`categories: [思考, 战略执行]`

## tags：3-5 个具体工具词/系列词

- 用具体名词：5S、VSM、方针管理、变化点管理、层级会议、指标树、标准化、AI、精益数字化、数据治理、现地现物、自働化……
- 系列标签：`影视精益课`（所有借电影讲精益的文章）
- **同义词合并规则**：策略部署/Hoshin → 方针管理；班前会 → 层级会议；禁用泛词（工具、转型、管理）
- 新增标签前先想：三个月后还会有第二篇文章用它吗？不会就换现有词

## 快查：新文章 front-matter 模板

```yaml
categories: [思考, 数字化与AI]
tags: [AI, 精益数字化, 标准化]
description: 一句话摘要（观点思考页/SEO/分享卡共用）
media_subpath: /assets/img/posts/<slug>
image:
  path: cover.png
  alt: 简短说明（勿含中文引号）
```

封面模板：`src/_cover-template/covers.html` + `render.cjs`（2.35:1 瑞士风，强调色按篇选）。
